<p><span style="font-weight: 400;">The rapid pace of modern software development, fueled by AI-driven coding and accelerated deployment cycles, has resurfaced a challenge that many development teams already struggled with: the speed of incident response must now match the speed of change. Every day, teams ship code faster than ever, which inevitably increases the risk of a new issue making it to production. The traditional approach—where engineers waste time jumping between disconnected tools—is no longer sustainable. It burns developers out and takes them out of a flow state. The solution is an interconnected AI ecosystem that leverages the operational data you already own.</span></p>
<p><span style="font-weight: 400;">PagerDuty is contributing to this interconnected AI ecosystem via </span><a href="https://modelcontextprotocol.io/docs/getting-started/intro"><b>Model Context Protocol (MCP)</b></a><span style="font-weight: 400;">, a standardized way for specialized AI tools to securely exchange information and actions. MCP acts as the common language, allowing AI tools and agents to talk directly to other tools and agents. </span></p>
<p><span style="font-weight: 400;">To date, we have over </span><a href="https://github.com/PagerDuty/pagerduty-mcp-server"><span style="font-weight: 400;">60 tools</span></a><span style="font-weight: 400;"> that allow users to pull in critical incident data, service information, and even trigger automated responses in any AI-enabled tool of choice. And we’re always adding more tools (check out our </span><a href="https://support.pagerduty.com/main/changelog"><span style="font-weight: 400;">release notes</span></a><span style="font-weight: 400;">). We plan to build out MCP parity with our open APIs, meaning that all the critical PagerDuty data and actions available via API will be available via MCP. The best part? This data can be used during incidents, or when coding. Let’s look at how a flow could work for each of these scenarios.</span></p>
<h3><span style="font-weight: 400;">Preventing Incidents with the Right Data at the Right Time</span></h3>
<p><span style="font-weight: 400;">Imagine you’re creating a new agent, perhaps one that guides your users through a check-out experience, offering deals they should add to their cart before the final purchase. This agent then could share this information back with other teams that are looking for demand signals on popular products. It’s critical that this agent both provides a good customer experience (relevant suggestions, works as intended) and that it relays the correct final purchase information back to internal teams. Now, imagine that you’re making a small tweak to this agent that should allow the user to rate the helpfulness of the agent’s suggestion. Let’s prevent a potential incident.</span></p>
<ol>
<li><b> Building Safer Agents with Past Incident Knowledge with LangSmith</b></li>
</ol>
<p><span style="font-weight: 400;">The </span><a href="https://www.langchain.com/templates/pagerduty-incident-response"><span style="font-weight: 400;">PagerDuty Incident Responder agent for LangSmith</span></a><span style="font-weight: 400;"> connects to the PagerDuty MCP server, accessing a service&#8217;s incident history and context. Developers can input a service name (such as the one this new agent is associated with), incident links for previous incidents with this agent, or symptom description from past failure. In response, PagerDuty will provide critical details that help developers assess risk: past incidents, triage information, and known failure modes discovered in post-incident reviews. This helps a developer prepare for a deploy with the right data at the right time.</span></p>
<ol start="2">
<li><b> Scoring Code Risk Before Deployment with Claude Code</b></li>
</ol>
<p><span style="font-weight: 400;">Developers who code in Claude Code can also score the risk of the uncommitted code changes right in their development workflow as another safety mechanism. The </span><a href="https://claude.com/plugins/pagerduty"><span style="font-weight: 400;">PagerDuty Plug-in for Claude Code</span></a><span style="font-weight: 400;"> is a risk scoring tool that brings production context directly into the development process. When a developer runs a simple command like </span><span style="font-weight: 400;">/risk-score</span><span style="font-weight: 400;">, Claude analyzes the new code against 90 days of PagerDuty incident data. The analysis identifies high-risk file types, the extent of the change, and whether it overlaps with areas that have caused past incidents. The developer then receives a clear risk score and actionable recommendations before the code is committed, helping to reduce the risk and cost of major operational failures.</span></p>
<ol start="3">
<li><b> Checking System Health Before Deployment with GitHub Copilot</b></li>
</ol>
<p><span style="font-weight: 400;">The </span><a href="https://github.com/github/awesome-copilot&amp;utm_source=partner-PagerDuty-pr-customagents-cta&amp;utm_medium=social&amp;utm_campaign=universe25"><span style="font-weight: 400;">PagerDuty Incident Responder custom agent for GitHub</span></a><span style="font-weight: 400;"> gives users access to PagerDuty data, including change correlation and incident data, directly within GitHub Copilot. Additionally, developers can build their own custom agents using PagerDuty MCP tools that offer even broader sets of data and actions. Users can quickly review what is currently happening in the system, ask about previous incidents on the service, and even summarize post-incident review notes. This can flag any concerns that may warrant postponing a deployment.</span></p>
<p></p>
<h3><span style="font-weight: 400;">Accelerating Response During an Incident</span></h3>
<p><span style="font-weight: 400;">The reality is that not every incident can be stopped, especially with the accelerated rate of shipping new code. When an incident does occur, MCP helps teams recover faster by reducing disruption and the cognitive load of having to jump between different tools. Let’s use our new agent example. Say the developer pushing the change to add the rating system skipped the review process above, and an issue slipped through the cracks. Here’s how MCP can make response smoother.</span></p>
<ol>
<li><b> Acknowledge and Review in Cursor</b></li>
</ol>
<p><span style="font-weight: 400;">When a new alert fires, you can immediately acknowledge and review it without leaving your coding tool. The PagerDuty MCP Integration with Cursor allows Cursor to pull in </span><a href="https://github.com/PagerDuty/pagerduty-mcp-server"><span style="font-weight: 400;">PagerDuty data or execute actions</span></a><span style="font-weight: 400;">, including who is currently on-call, service status details, and incident history. This can help a developer answer key questions and begin triage, asking questions to PagerDuty about incident impact and services, any notes that are pre-populated, and more. Without context switching, a user could also ask GitHub Copilot about recent changes, bringing that information in-line with the critical PagerDuty data without ever leaving their tool of choice.</span></p>
<ol start="2">
<li><b> Automated Diagnostics and Suggested Fixes with Honeycomb data</b></li>
</ol>
<p><span style="font-weight: 400;">While a developer is reviewing the issue, the </span><a href="https://www.pagerduty.com/platform/ai-agents/sre/"><span style="font-weight: 400;">PagerDuty SRE Agent</span></a><span style="font-weight: 400;"> is running diagnostics in the background. PagerDuty will be extending its SRE Agent to use logging and metrics data from Honeycomb via MCP. The SRE Agent will use this critical telemetry to inform triage, quickly determine the root cause, and execute more pointed automation, taking the initial diagnostic burden off the human responder. For example, the agent can quickly suggest a fix, like rolling back a recent change.</span></p>
<p></p>
<ol start="3">
<li><b> Quick Fix and Resolution</b></li>
</ol>
<p><span style="font-weight: 400;">Thanks to this seamless flow of information, the responder can then go back to Cursor to take the suggested action—like rolling back the change. This unified, intelligent workflow quickly closes the loop from alert detection to resolution without pushing a user to a different surface. Response is faster, and developers can get back to building with less time spent on interrupt work.</span></p>
<p><span style="font-weight: 400;">By connecting data and actions from tools like LangSmith, Claude, GitHub Copilot, Cursor, Honeycomb, and more, PagerDuty is making the right data and actions accessible exactly where teams need it. This approach helps reduce friction, accelerate incident management to match the pace of AI-driven development, and ultimately gives developers more time back for higher-value work. We are only scratching the surface of what is possible with MCP.</span></p>
